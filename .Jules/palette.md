## 2024-03-04 - Improve Keyboard Accessibility for Hover-Only Overlay Elements
**Learning:** Common design patterns often use container elements with `opacity-0` and `group-hover:opacity-100` to show controls only when a mouse hovers over them (e.g., video player controls, carousel arrows). This pattern completely hides the elements from keyboard users because tabbing into them does not trigger the `hover` state, and they remain invisible despite receiving focus.
**Action:** When creating or fixing hover-only control overlays, always include `focus-within:opacity-100` on the container (or `focus:opacity-100` / `focus-visible:opacity-100` on the individual interactive elements) to ensure the controls become visible when a keyboard user tabs into them.
